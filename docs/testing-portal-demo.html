Forge — Testing Portal

A complete tester experience so authorized users can validate every workflow, screen, integration, role and domain before any production implementation.

What it is

A guided, sequential testing experience built into the app. Authorized testers sign in, open the Testing Portal, and validate each domain step-by-step — with a clear instruction, the expected result, and success criteria for every step. Progress is tracked, can be resumed, and issues can be reported inline.

Who can test

Access is restricted to a fixed list of accounts (gated by email). One Testing Admin controls the environment.

The flow

  1. Open the Testing Portal from the sidebar. See the four domains — AIMLDVASDSE — each with a status: Not StartedIn ProgressCompleted and a progress bar.
  2. Start Testing a domain (becomes Resume Testing once begun). The domain auto-populates a realistic environment (teachers, mentors, team leads, students, teams, repositories, milestones, deliverables) using the real tester accounts.
  3. Work the guided steps in sequence. Each step tells you what to do, what to expect, and how to know it passed.
  4. Mark each step Done or Skip; Report Issue any time. Progress saves automatically.
  5. When every step is handled → Domain Completed.

Team → shared repo → deliverables + engineering work. Validate the shared repo, milestones, contributions.

Role-specific testing

Every domain includes test cases for each role: Admin (user management, invitations, domains, integrations), LCC (onboarding, communications, teams), Teacher (domain analytics, team & progress tracking), Mentor (repository tracking, reviews, contribution monitoring), Team Lead (team management, deliverables, repository monitoring), and Mentee (repository, tasks, milestones, submissions). Plus integration checks — GitHub, Discord, Calendar, Notifications. (See role-test-cases.md.)

Issue reporting

On any step, Report Issue opens a form: Title · Description · Severity (Low/Medium/High/Critical) · screenshot placeholder. Reported issues appear in the domain's Reported issues panel.

In presentation mode reports are recorded locally. In production they email shaik.tajuddin2024@nst.rishihood.edu.in.

Progress & resume

Each step is tracked as done / skipped / current. Domain status is derived (Not Started → In Progress → Completed). Leave and return any time — the domain reopens where you left off, and completed domains can be reset to run again.
