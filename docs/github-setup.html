GitHub — org App, webhooks & per-user connect

Set up the org-owned GitHub App for @lcc-ai-nst: one installation reads all repo activity, and a lightweight per-user OAuth links each Forge user to their GitHub identity. Every callback & webhook points at the server (Express), never the client. Enter your production domain and the URLs update automatically. ← back to all integrations

Local dev always uses http://localhost:4000 (server). Prod becomes https://app.example.com.

⌥ GitHub — App + webhooks

Use a GitHub App (org-level, fine-grained, higher rate limits) to read repo activity and receive webhooks.

Two-token model. The App is installed once on the @lcc-ai-nst org → its installation token reads all org repo activity (commits, PRs, reviews, issues). Each person's “Connect GitHub” is only a per-user OAuth to learn their GitHub id + username, so org activity can be attributed to the right Forge user. You do not store a token per user — the org install does the data fetching.
  1. Create the App under the organization (so it's owned by the org, not your personal account) — you must be an org owner: open github.com/organizations/lcc-ai-nst/settings/apps/new. GitHub App name: forge-dev (dev) / forge (prod) — names are globally unique.
  2. Homepage URL — this is the client/UI (the only field that points at the client, not the server):
    Productionhttps://app.example.com
    Local devhttp://localhost:3001
  3. Callback URL (server OAuth callback — for per-user repo linking):
    Production
    Local devhttp://localhost:4000/api/integrations/github/callback
    Leave Setup URL blank. Keep Expire user authorization tokens on (gives a refresh_token). Leave Device Flow off; check Request user authorization (OAuth) during installation only if you want the per-user link flow.
  4. Webhook → keep Active. Generate a strong Webhook secret with openssl rand -hex 32 and paste the same value into GITHUB_WEBHOOK_SECRET.
    Production
    Local devhttps://smee.io/<your-channel>
    GitHub can't reach localhost — webhooks POST from the public internet. For local dev, create a channel at smee.io/new, use that URL as the Webhook URL, and forward it to your server:
    npx smee-client --url https://smee.io/<your-channel> --target http://localhost:4000/api/webhooks/github
    (ngrok works too: ngrok http 4000 → https://<id>.ngrok.io/api/webhooks/github.)
  5. Repository permissions:
    PermissionAccess
    ContentsRead-only
    IssuesRead-only
    Pull requestsRead-only
    MetadataRead-only
    Organization permissions: Members → Read-only.
  6. Subscribe to events: pushpull_requestpull_request_reviewissuesmilestone (the auto-suggested Installation target / Meta / Security advisory events aren't needed)
  7. Where can this GitHub App be installed? → Only on this account (@lcc-ai-nst). It's an internal org app — there's no reason to let other orgs install it. (Any account is only for public/marketplace apps; don't pick it.)
  8. Create the App → Generate a private key (downloads a .pem) → note the App ID.
  9. Install the App on the organization (this is what produces the installation token used to read all org activity):
    1. On the App's page → Install App → choose @lcc-ai-nst → Install.
    2. Repository access: pick All repositories (recommended — new team repos are covered automatically) or Only select repositories.
    3. After installing, the URL is …/installations/<installation-id> → copy that Installation ID.
  10. Allow it org-wide: in Org → Settings → Third-party access, make sure OAuth app access isn't blocking your app, so members can complete the per-user “Connect GitHub” step. Org owners may need to approve it once.
Env you get → server/.env
GITHUB_APP_ID · GITHUB_APP_PRIVATE_KEY · GITHUB_APP_INSTALLATION_ID · GITHUB_WEBHOOK_SECRET
The server verifies every webhook with the X-Hub-Signature-256 HMAC against GITHUB_WEBHOOK_SECRET — required, don't skip it.
