Forge Landing Page Storyboard LCC · NST

Forge marketing landing — section-by-section storyboard

A scroll-through design reference for the public landing page. Each frame below maps one section of the page in order — its purpose, hero visual, copy gist, and on-scroll motion — so a stakeholder can grasp the whole narrative in one pass. Forge is the Profile Building Drive platform built by the Learner Career Council at Newton School of Technology: students build, mentors guide, teachers track, and learning becomes measurable through real GitHub contribution.

Trust · it's an institution platform

Security & governance

Reassure decision-makers: access is controlled, auditable, and isolated.

Key visual — layered guard

Concentric guard layers from login down to the audit trail.

Content summary

Built like institutional software — because it is.

Google-only login restricted to the NST hosted domain and an invite allowlist; server-side RBAC enforced at three layers; signed webhooks; an immutable audit log; and a dedicated, isolated AWS footprint.

14

Under the hood · for the technical reader

Platform architecture

Show the clean client → server → data topology on AWS.

Key visual — topology diagram

Next.js client → Express server → Postgres + Redis on AWS ECS.

Content summary

Thin client. Strong server. One clean boundary.

A Next.js UI talks only to an Express API that owns all logic, integrations, and webhooks, backed by Postgres + Redis. Runs on AWS ECS Fargate behind an ALB, isolated to its own VPC.

Key visual
A left-to-right topology diagram: client box → server box → data stores, with the ECS/ALB context noted.
Animation on scroll
Boxes appear in request order; connectors draw between them; a request "packet" animates client → server → DB and back to show the flow.
